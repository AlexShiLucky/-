重要な情報

﻿
  • EWARM 8.20以前からの特定のリリースノートのドライバ
    • IAR C-SPY I-jetおよびJTAGjetドライバ
    • IAR C-SPY J-Linkドライバ
    • IAR C-SPY Texas Instruments Stellaris FTDIドライバ
    • IAR C-SPY GDBサーバドライバ
    • IAR C-SPY ST-Linkドライバ
    • IAR C-SPY XDSドライバ
    • IAR C-SPY MSP-FETドライバ

  • マルチコアサポートには、I-jetデバッグプローブの1つまたはCMSIS-DAPインターフェースが必要です。

  • ARMv8-Mシミュレータは、セキュアモードおよび非セキュアモードを完全にサポートしていません。

  • assert()の新しい動作
    (7.10.1の新機能)assert()関数でダイアログボックスが表示され、ユーザが中止、デバッグ、無視を選択できるようになりました。
    以前の動作を希望する場合は、アプリケーションのソースコードに以下を追加します:

    void __aeabi_assert(char const * msg, char const *file, int line)
{
printf( "%s:%d %s -- assertion failed\n", file, line, msg ); 
abort();
}

  • C-SPYマクロ__loadModule(filename, suppress)は、__loadImage(patch, offset, debugInfoOnly)という名前に変わりました。

  • C-SPYマクロ__sleep(microseconds)は、__delay(milliseconds)に置き換えられました。時間の間隔の違いに注意してください。

  • ARM用IAR Embedded WorkbenchのC-SPYには、フラッシュローダ機能のための改善されたメカニズムと設定方法が用意されています。以前の設定のフラッシュローダを採用する既存のプロジェクトは、自動的にデフォルト設定を持つ新しい方法に移行されます。ほとんどの場合、デフォルト設定で十分です。以前の設定は新しい設定ファイル(拡張子.board)に変換され、デフォルト設定が適切でない場合に使用することができます。フラッシュローダの新しいメカニズムおよび設定方法の詳細は、フラッシュローダガイドを参照してください。

  • mbedデバイス(例えば LPC1768)を装備したCMSIS-DAPを使用する前に、テクニカルノート69741をお読みください。

その他

﻿
  • APB/AHBアクセスポートがメモリゾーンとして使用可能に

    MEM-APアーキテクチャを実装したAPB/AHBアクセスポートが、メモリゾーンとしてアクセス可能になり、カスタムのDDFファイルを使用して[メモリ]ウィンドウ、[デバッガ]マクロウィンドウ、[レジスタ]ウィンドウからアクセスすることができます。

    ゾーン名は以下のとおりです:

    • AP_Memory: 現在選択されているポート。

    • AP0_Memory - AP5_Memory: 0 ~ 5個のポートを表示します。

    これらのゾーンへのアクセスの動作は、選択したポートが存在しない、MEM-APアーキテクチャでない、他の理由でアクセスできないといった場合に、予測不可能となることがあります。
    現時点では、これらのゾーンはC-SPY I-jetドライバまたはC-SPY CMSIS-DAPドライバの使用時にのみアクセスすることができます。

新機能

  • 向上したコードカバレッジとプロファイル機能のためのストリーミングトレース
    IAR I-jet Traceプローブを使用している場合は、C-SPYのコードカバレッジとプロファイルウィンドウに、ストリーミングトレースデータをフィードできます。これによりコードカバレッジとプロファイリングデータの両方を無限テスト実行中に収集できるようになりました。詳しくは、IAR Embedded Workbench C-SPYデバッグガイドを参照してください。

  • 拡張したマルチコアサポート
    マルチコアサポートは、任意の数の非対称コアをサポートするように拡張されました。例えば、1つのA9、1つのM7、2つのM4コアのあるデバイスをデバッグして制御できるようになり、デバッガコントロールのArm Coresight CTIメカニズムですべて同期されます。

  • パフォーマンスモニタウィンドウ
    パフォーマンスモニタウィンドウは、CortexAおよびCortexRプロセッサで使用可能なパフォーマンスモニタユニット(PMU)を介した、イベント回数やCPUクロックサイクルを表示します。この機能はIAR I-jetプローブを使用しているときのみ使用できます。ライブ更新は最近のすべてのCortex-Aコアで使用できます。詳しくは、IAR Embedded Workbench C-SPYデバッグガイドを参照してください。

  • Remedy RTOS Viewer
    RowebotsのRemedy RTOSには、C-SPY RTOS認識プラグインがデバッガに含まれるようになりました。
